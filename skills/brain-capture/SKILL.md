# brain-capture — append the WHY (the gold code can't regenerate)

Captures the *delta of intent/decisions* — the durable WHY. **Architecture (the HOW) is NOT captured here**; it is a regenerated local cache (`/brain-protocol:brain-sync`). This skill only writes the **committed** WHY layer: `decisions/` + (rarely) `product/`. Runs **in-session on this chat's own model** (latest, best — every stage).

## Steps
1. **Promote the inbox.** Read `.claude/brain/_inbox.md` (per-response harvester output, local/unreviewed). Promote each real, durable item to an ADR (step 2); discard chit-chat/duplicates. Delete promoted/rejected lines from `_inbox.md`.
2. **Write ADR(s)** in `.claude/brain/decisions/NNNN-<slug>.md` (next number). Each ADR is self-contained:
   - **Date** (ISO), **Commit** (short SHA), **Status** (`accepted` | `superseded by NNNN`), **Touches** (files/areas).
   - **Decision** — what was decided / learned / corrected.
   - **Why** — reasoning, constraints, alternatives rejected.
   - **Implications for the brain** — which `product/` or `architecture/` entries to update; link related entries with `[[name]]`.
   - If an ADR overrides an earlier one, mark the old one `Status: superseded by NNNN` — never move/delete it.
3. **Rebuild the effective digest incrementally.** In `decisions/CURRENT.md`, fold in only ADRs numbered above its `through:` watermark, applying supersessions (drop overridden decisions — they stay in the log). Set `through:` to the highest ADR number.
4. **Update the durable layer ONLY if intent changed.** A real product/business decision updates `product/*.md` — confirm material intent shifts with the human first. Code-derivable facts do NOT go here; they belong in `architecture/` (regenerated by `/brain-protocol:brain-sync`).
5. **Update INDEX.md** — one line per new ADR.
6. **Boundaries.** Do **not** regenerate `architecture/` here (that's `/brain-protocol:brain-sync`). **Write-only — never commit or stage.** Leave the WHY in the working tree; the user commits it (so the team gets it on pull).

## Invariants
- Delta, not re-summary. Append-only ADR log (supersede, never move/delete). CURRENT.md rebuilt incrementally from the log. The HOW is regenerated elsewhere, never captured here. Never auto-commit.
